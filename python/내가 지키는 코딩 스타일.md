

# 2. 클래스, 객체(인스턴스)

- class 는 CamelCase 활용. (관례)

- 내부적으로만 사용하는 인스턴스 변수는 __ 를 붙임. (관례)

  ```python
  class MyClass:
      def __init__(self, age):
          self.age = age
          self.__private = age * 10
  ```

- 클래스가 적용된 객체는 snake_case 와 CamelCase를 합침. = Snake_Camel_Case
  일반적인 파이썬 스타일가이드에서 추천하지 않는 방식이지만 순수 클래스와 순수 변수와는 다르게 클래스가 적용된 변수(객체) 라는 점을 드러내기 위함.

- 단, 기존의 내장 클래스가 CamelCase로 만들어져있지 않아 보통의 모듈하고 구분이 되지 않을 경우는 그냥 단순 변수 방식으로 생성.

- 마찬가지로 기존의 내장 모듈이 CamelCase로 만들어져있다고 하더라도 객체 방식으로 생성.

  ```python
  Kim_Yoon_Hyun = MyClass(10)
  ```

# 3. 연산자

- 우선순위가 높은 연산자를 우선적으로 붙여서 씀

- 단, 현재로썬 복잡하게 식이 들어가있는 경우에 대해서의 규칙이 명확하게 잡혀 있지 않음. 
  -> 우선적으로 가독성이 좋아보인다고 보이는 형태로 설정.

  ```python
  a = 1
  b = 2
  c = 3
  num = a * b + a / c # X
  num = a*b + a/c  # O
  
  # 아래는 가독성 여부를 주관적으로 판단
  num = (a*b + a/c) * (a%c)
  num = (a*b + a/c)*(a%c)
  num = (a*b+a/c) * (a%c)
  ```

## 4.1. = 관련

- 함수, 클래스 등의 *option 설정시에는 = 사이에 띄어쓰기 하지 않음*
  ```python
  variable = my_function(a, b, option1 = True, optione2 = False)  # X
  variable = my_function(a, b, option1=True, optione2=False)	# O
  ```

## 4.2. , 관련

- *, 뒤에는 반드시 띄어쓰기 적용*

  ```python
  variable = my_function(a,b,option1=True,optione2=False)		# X
  variable = my_function(a, b, option1=True, optione2=False)	# O
  ```

## 4.3. 변수 나열 방식

### 4.3.1. 함수 생성시

- 함수 입력 변수 나열시 우선순위가 높은 변수를 앞에 적음

```python
def temp(option1, data):	# X
def temp(data, option1):	# O
```

- 주관적 판단 하에 입력 변수가 너무 많을 경우 줄바꿈으로 조절
- 상세 규칙은 아래와 같음

```python
# 정상 예시
def job(df, date_, valid_col_list, column_data, # 함수명 및 ( 이후 변수를 바로 붙여 적음
         nan_repeat, soot_max_diff, corr_value, # 첫 줄바꿈시 적용되는 위치를 기준으로 계속 줄바꿈
         burn_stan, stop_stan, 	# 변수의 줄바꿈 기준은 변수간의 상관성을 주관적으로 판단함
         bus_name, plot_col_dict, 
         save_path=None):  # ): 로 마무리 할때는 줄바꿈을 하지 않음
    
# 잘못된 예시
def temp(
    data_var1, data_var2,   # ( 이후에 줄바꿈이 적용됨
    	time_var1, time_var2, 	# 줄바꿈의 위치가 어긋남
    time_var2, df_var1, # 변수간의 상관성이 어긋남
): # ): 로 마무리 시 줄바꿈이 적용됨
```

### 4.3.2. 함수 선언시

- 입력변수는 2개 이상시 반드시 한 변수당 한번의 줄바꿈을 적용함

- 상세 규칙은 아래와 같음

```python
def temp(var1, var2, option1=True):
    return r1, r2

# 정상 예시
result1, result2 = temp(	# 첫 입력 변수는 줄바꿈 이후 적음
    var1=1, 	# 줄바꿈 위치는 함수명 위치와 상관없이 항상 '출력변수위치+들여쓰기한번' 의 위치에 설정
    var2=2, 	# 모든 입력 변수에 대해 변수명=입력변수 형태로 적음
    option1=False
)	# ) 로 마무리 시 출력변수의 위치에 줄바꿈 적용


# 잘못된 예시
result = temp(2, 1, False)  # 입력할 변수를 '변수명=입력변수' 형태로 하지 않음
result = temp(var1=1, var2=1, option1=False)  # 입력변수에 대해 줄바꿈을 적용하지 않음
result = temp(var1=1,	# 첫 입력변수에 줄바꿈을 적용하지 않음
             var2=2,
             option=False)	# ) 마무리 시 줄바꿈을 적용하지 않음
result1, result2 = temp(	
                        var1=1,		# 변수 줄바꿈 위치가 출력변수 기준 들여쓰기 한번으로 적용되지 않음
                        var2=2,
                    )
```

- 들여쓰기 조절시 변수는 항상 함수명 아래에 들여쓰기 한번의 위치에 존재함
- 마지막 ) 은 항상 단독으로 모든 변수명 아래 함수명과 같은 들여쓰기 위치에 존재함

```python
def temp(var1, var2, option1=True):
    return None
```
