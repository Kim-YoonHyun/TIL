1. 타인의 레파지토리 fork

2. clone

   최초

   ```bash
   git clone https://github.com/your-username/forked-repo-name.git
   ```

   최초 이후

   ```git
   git pull origin main
   ```

3. 브랜치 생성

   ```bash
   git checkout -b new-feature-branch
   ```

4. 수정 및 커밋 진행
   ```bash
   git add .
   git commit -m "Add new feature or fix"
   ```

5. 새로만든 branch 에 푸시
   ```bash
   git push origin new-feature-branch
   ```

6. pull request 요청
   정상적으로 새로운 브랜치에 커밋이 진행 되면 compare & pull request 버튼이 생김. 해당 버튼을 통해 pull request 생성 진행

이때, 하나의 브랜치에서는 하나의 PR 만 생성가능하기에

새로운 PR 을 하려면 브랜치를 새로 만들어야함
